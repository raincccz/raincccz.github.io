hexo写作说明

hexo核心文件夹

仓库名

- source（存放源文件）
-- _posts（存放post类型即发布文件
-- drafts（存放草稿）
- themes（存放主题文件）
- scaffolds（存放

post文件头实例

文件头内容

  1. layout
  2. title
  3. date
  4. categories(复数）
  5. tags（复数）

一些说明

文件说明

  1. 文章标题尽量与文件名一致（不严格要求）（文件名需要与文件夹名一致）
  2. 不需要重复写文章标题（# 标题），避免冗余以及文章标题时多次修改
  3. 草稿存在drafts目录下，需要发布时移动到post文件夹
  4. 发布的内容存在source下
  5. 文件创建，使用hexo new post article_name
  6. post文件夹下可以建立文件夹，对其他结构无影响

文件头说明

  1. 文件头冒号后需要加空格
  2. categories确定大致分类
  3. tags确定文章内容 使用多个tags时，使用列表格式书写
  4. post类型文章需要写layout，drafts可以省去layout
  5. typora-root-url: 设置图片根目录（图片保存在同名的文件夹）

文件内容说明

  1. 用<!-- more -->来标记文章正文

  2. markdown语法 标题井号后需要加空格

  3. 图片可以使用相对路径，也可以使用绝对路径（根目录是source文件夹）
    内容保存说明

  4. 不需要写目录，有hexo主题生成目录，编辑时使用typora生成主题
